Scroll To Show Component


What IS Scroll To Show Component ?

With the Scroll To Show component, display page elements only when the user reaches them and make your page more attractive. It is set up with just a few lines of code and works well anywhere

call scroll to show Component 

import { EasyJS } from '/components/dist/easyjs.js'; 
    EasyJS.component('scrollToShow', {
        targetElements: '.label-an',
        animationDelay: '1s',
        animationIn: 'fadeInUp',
    });
    EasyJS.component('scrollToShow', {
        targetElements: 'pre , .scrollAreaNote',
        animationIn: 'bounceIn',
        animationSpeed: '2s',
        threshold: 0.7,
        animationDelay: '0.5s'
    });
    EasyJS.component('scrollToShow', {
        targetElements: 'p',
        animationIn: 'fadeIn',
        animationSpeed: '6s'
    });
    EasyJS.component('scrollToShow', {
        targetElements: 'section li',
        animationIn: 'fadeInTopRight',
});

How Its Work ?

Honestly, working with the SCROLL TO SHOW Component is easier than it seems, let me explain : In the code above I create 4 scroll to show component, then with targetElements option select scroll to show targets. (this page is animated with above code)

SO

The basic usage is something like this 

import { EasyJS } from '/components/dist/easyjs.js'; 
EasyJS.component('scrollToShow', {
   targetElements: '', // your target elements or target selectors
});

Use onScreen And outScreen Callbacks

These two help you manage the elements in the viewport carefully 

import { EasyJS } from '/components/dist/easyjs.js'; 
EasyJS.component('scrollToShow', {
    targetElements: 'p',
    onScreen : function(element) {
       element.style.background = 'green'; // set green when current element is in viewport
    },
    outScreen : function(element) {
       element.style.background = 'red'; // set red when current element is out viewport
    }
});

Usage scrollArea option Sometimes IS important !

When the target elements are displayed in window, this works fine. However, if the target elements are located within an overflow area and the user needs to scroll to view them, you need to set the scrollArea option according to your scroll area to ensure it works properly

scroll to show Component Options

on below part you can see total scroll to show Component Options

@ Required Options

  • targetElements : string () => list of target elements or selectors

@ Optional Options

  • threshold : number (1) => threshold of target elements to show
  • animationIn : string (fadeInDown) => target ELM animation in
  • animationOut : string (fadeOut) => target ELM animation out
  • animationSpeed : string (2s) => modal target ELM animation speed
  • animationDelay : string (0s) => target ELM animation Delay
  • onScreen : callback(TargetElement) => fired when target elements is inside viewport
  • outScreen : callback(TargetElement) => fired when target elements is outside viewport
  • scrollArea : element| window | object (winow.document) => viewport scroll area
